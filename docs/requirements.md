# 视频去重工具 - 开发需求与规则说明 (requirements.md)

本文件定义了视频去重工具（quchong）的业务需求和开发规则。

## 1. 业务目标
帮助用户在 Windows 系统下快速检索指定文件夹（含子文件夹）中的视频文件，识别出 FC2 视频和日本 AV 番号视频，自动过滤文件名中的广告网址等垃圾信息，找出其中的重复文件，并在界面中进行分类聚集展示，供用户手动或一键安全删除（移至回收站）。

## 2. 核心需求列表

### 2.1 目录检索与文件过滤
*   支持拖拽文件夹到软件界面，或通过按钮调出系统窗口选择文件夹。
*   拖拽完成后应能自动启动检索，提升用户体验。
*   扫描范围应包括子文件夹中的所有常见格式视频：`.mp4`, `.mkv`, `.avi`, `.wmv`, `.mov`, `.flv`, `.rmvb`, `.m4v`, `.ts`, `.3gp`, `.mpeg`, `.mpg`。

### 2.2 文件名清洗与特征值提取 (Parser)
*   文件名多变，需过滤掉垃圾网址（如 `hjd2048.com`、`www.t66y.com_` 等）、中括号标记（如 `[1080p]`、`[ThZu.Cc]`）以及其他无关信息。
*   **汉化标注处理**：类似于 `-C`、`_字幕`、`_ch`、`_汉化` 等汉化标记，仅在提取对比特征码时过滤，但最终界面文件列表展示、定位和删除操作中必须保留其原始文件名。
*   **分段视频区分 (-1/-2/part1/CD1)**：对于同一视频的不同分段，系统在提取唯一 ID 时会自动解析并保留其分段后缀（格式化为 `-1`、`-PARTA` 等），**避免把同视频的不同分段判定为重复视频而误删**。
*   **特典与 SP 视频区分**：解析特典或特别篇标识（如 `_sp1`、`_sp2`、`特典`、`BONUS`）。SP 视频与正片会以不同的 Key 相互独立区分，防止误删，只有 SP 自身重复时才会被聚合去重。
*   **普通视频降级查重 (Fallback)**：如果视频不含标准的 AV 番号或 FC2 特征，则采用降级清理，提取剔除中文字幕标识、括号、空白符号后的中文/英文纯文件名核心作为特征值。只有过滤出的一般视频名称完全一致时，才作为重复项呈现。忽略通用占位名（如 `SAMPLE`、`INTRO`）以及短名称。
*   清洗后提取出唯一的特征码：
    *   **FC2 系列**：规范化为 `FC2-XXXXXX[-分段/SP]`。
    *   **常规番号**：规范化为 `大写字母-数字[-分段/SP]` (如 `SSIS-023-1`、`ABP-456-SP1`)。
    *   **普通视频降级特征码**：纯字母、数字、中文的核心特征字符串。

### 2.3 重复判定与数据聚合
*   以提取出的唯一特征码作为 Key。
*   **广告推广视频聚类**：当视频不含任何番号但其名称带有广告词（如 `社区最新情报`、`地址发布`、`永久域名` 等）时，自动将其统一归类到 `[ADVERTISEMENT]`（广告宣传视频）分组中，不受上述番号去重规则限制。
*   当一个 Key 关联了 2 个及以上的视频文件路径时，判定为重复组（广告视频组即使不重复也全部罗列在一起）。
*   重复组内的文件应能读取并显示辅助判定信息：
    *   文件大小（格式化为 MB/GB）。
    *   文件修改时间。
    *   视频分辨率与时长（如可提取）。
*   重复组列表在 UI 中按该组**重复文件的个数**降序排列，且大文件组优先展示。

### 2.4 清理与删除机制
*   **单个移至回收站**：在每条文件记录后提供一个垃圾桶按钮，点击后将该文件送入 Windows 回收站，安全防误删。
*   **一键去重**：在每个重复组的卡片头部提供“一键去重”按钮。点击后，系统通过算法自动识别该组内的“最佳文件”（文件体积最大、分辨率最高者），保留该文件，并自动将该组内的其他重复文件全部移至回收站。
*   **广告视频的一键清理**：广告宣传视频卡片特制为红色警示边框，其浪费空间显示为该组内**所有文件的体积之和**。点击 **“一键清理广告”** 时，系统会**全量移入回收站（不保留任何副本）**。
*   **打开定位**：提供文件夹图标按钮，点击后在 Windows 资源管理器中打开并高亮选中该视频文件。

---

## 3. 开发规则与红线
1.  **安全删除**：必须调用 Electron 提供的 `shell.trashItem` 接口将文件移至回收站，**绝对禁止**使用 `fs.unlink` 永久物理删除文件，以确保用户数据安全。
2.  **异步响应**：文件读取及大目录递归扫描必须异步处理，不得阻塞 Node.js 事件循环，确保扫描过程中 GUI 界面不卡死且可以随时取消。
3.  **UI 降级体验**：当无法成功提取视频的时长/分辨率时（例如非 Chromium 兼容的 avi 格式且 ffprobe 不可用），程序不能报错退出，应优雅降级仅展示文件大小和修改日期。
