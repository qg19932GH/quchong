# Agent Custom Rules

1. **先确认、后编码**：在执行任何涉及代码修改或生成的任务前，必须先将设计思路、修改逻辑或实现计划向用户清晰陈述，取得用户的明确许可后，方可动笔编写代码或执行更改。
2. **多方案时必须提供选择**：当一个功能或问题修复有多种实现方案或设计方案时，机器人必须以选择题或对比列表的形式向用户提供选项，由用户决定最终的方案，不得单方面直接决定并直接编写代码。
3. **推送与编译规范**：向 Git 仓库 Push 提交时，Commit 消息必须使用中文，并且在消息中明确说明本次改动具体发生在哪些文件（例如：`feat: 镜像文件夹合并清理功能 (改动在 src/main.js, src/renderer.js)`）。
